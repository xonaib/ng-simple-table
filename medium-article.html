I got tired of rebuilding the same Angular table every project, so I made a library

Every Angular project I've worked on has a table. Usually more than one. And every single time, I end up writing the same boilerplate — MatTableDataSource, paginator, sort, filter inputs, loading state, column definitions... it's a lot of setup for something that should be straightforward.

After doing it the fourth time, I decided to just extract it into a library.

It's called ngx-mat-simple-table. It wraps Angular Material's mat-table in a clean, declarative API so you get sorting, filtering, pagination, multi-select, row actions, and more — without writing a wall of template code every time.

The problem with mat-table

Angular Material's table is actually pretty powerful. But it's also very low-level. You define every column manually, wire up your own paginator, create your own filter logic, manage selection state yourself... and most of the time you're just doing the same thing you did in the last project.

Here's roughly what a "basic" table with sorting, pagination, and a search filter looks like out of the box:

<mat-form-field>
  <input matInput (keyup)="applyFilter($event)" placeholder="Search" />
</mat-form-field>

<table mat-table [dataSource]="dataSource" matSort>
  <ng-container matColumnDef="name">
    <th mat-header-cell *matHeaderCellDef mat-sort-header>Name</th>
    <td mat-cell *matCellDef="let row">{{ row.name }}</td>
  </ng-container>
  <!-- ...repeat for every column... -->
  <tr mat-header-row *matHeaderRowDef="displayedColumns"></tr>
  <tr mat-row *matRowDef="let row; columns: displayedColumns;"></tr>
</table>

<mat-paginator [pageSizeOptions]="[10, 25, 50]"></mat-paginator>

And that's before you add selection, column-level filters, sticky headers, or a loading indicator. It compounds fast.

What simple-table looks like instead

With ngx-mat-simple-table, the same table is:

<simple-table
  [dataSource]="tasks"
  [tableColumns]="columns"
  [tableConfig]="config"
>
</simple-table>

You define your columns once in the component:

columns: ColumnDef[] = [
  { key: 'select' },  // built-in checkbox column
  { key: 'id',       label: 'ID',       width: 72,  sticky: 'left' },
  { key: 'title',    label: 'Title',    width: 220, hasColumnFilters: true },
  { key: 'status',   label: 'Status',   width: 140, hasColumnFilters: true,
    displayValue: v => String(v).replace(/-/g, ' ').toUpperCase(),
    cellClass: v => `status-${v}` },
  { key: 'priority', label: 'Priority', width: 110, hasColumnFilters: true },
];

config: TableConfig = {
  isPaginated: true,
  paginationOptions: { defaultPageSize: 25, pageSizeOptions: [10, 25, 50] },
  horizontalScroll: true,
  fillContainer: true,
};

That's it. Sorting, per-column dropdown filters, pagination, and row selection are all wired up automatically.

Custom cells when you need them

The declarative API covers most cases, but sometimes you need custom rendering — a link, a badge, a formatted date. You can drop in a template for any column without losing the auto-generated header (so mat-sort-header still works):

<simple-table [dataSource]="tasks" [tableColumns]="columns" [tableConfig]="config">
  <ng-template cellDef="title" let-row>
    <a [routerLink]="['/tasks', row.id]">{{ row.title }}</a>
  </ng-template>
</simple-table>

Everything else stays automatic. You only opt in to custom rendering where you actually need it.

Declarative actions

One pattern that keeps coming up in data tables is action buttons — a toolbar "Add" button, per-row edit/delete, a bulk-delete button that's disabled until rows are selected. The v1.2 release adds a declarative TableAction[] API that covers all of these with a single input.

Each action has a position that controls where it renders:

  • toolbar — left side of the toolbar row, next to the column chooser and export icons
  • above — same toolbar row, rendered first (useful for primary CTAs)
  • below — left side of the paginator row, so bulk actions sit right next to page controls
  • row-inline — icon button visible on every row in a sticky-end column
  • row-menu — items inside the ⋯ overflow menu in the same sticky column
actions: TableAction<Task>[] = [
  {
    id: 'add',
    label: 'New task',
    icon: 'add',
    position: 'toolbar',
    color: 'primary',
    variant: 'flat',
    cb: () => this.openCreateDialog(),
  },
  {
    id: 'bulk-delete',
    label: 'Delete selected',
    icon: 'delete_sweep',
    position: 'toolbar',
    color: 'warn',
    variant: 'stroked',
    disabled: () => this.selectedRows().length === 0,
    cb: () => this.bulkDelete(this.selectedRows()),
  },
  {
    id: 'edit',
    icon: 'edit',          // label omitted → icon-only button with tooltip
    position: 'row-inline',
    cb: (row) => this.openEditDialog(row),
  },
  {
    id: 'delete',
    label: 'Delete',
    icon: 'delete',
    position: 'row-menu',
    color: 'warn',
    cb: (row) => this.deleteTask(row),
  },
  {
    id: 'export-selected',
    label: 'Export selected',
    icon: 'file_download',
    position: 'below',
    disabled: () => this.selectedRows().length === 0,
    cb: () => this.exportSelected(),
  },
];
<simple-table
  [dataSource]="tasks"
  [tableColumns]="columns"
  [tableConfig]="config"
  [actions]="actions"
  (selectionChange)="selectedRows.set($event)"
>
</simple-table>

The label field is optional — omit it and the button renders as an icon-only mat-icon-button with a tooltip. For non-row positions you can also set variant to 'flat', 'stroked', or 'text' to control the Material button style. Row-inline actions are always icon buttons; row-menu items are always mat-menu-item.

Server-side mode

For larger datasets you'll want server-side pagination, sorting, and filtering. The component handles that too — you pass your current page slice as [dataSource] and wire up the events:

<simple-table
  [dataSource]="currentPage()"
  [length]="totalRows()"
  [pageIndex]="_pageIndex()"
  [tableColumns]="columns"
  [tableConfig]="config"
  (page)="onPage($event)"
  (sortChange)="onSort($event)"
  (filterChange)="onFilter($event)"
  (refresh)="onRefresh()"
>
</simple-table>

Each event gives you the current state so you can build your API params and feed the response back in.

If your API doesn't return a total row count — cursor-based pagination, for example — just omit [length]. The component will infer next/prev availability from the number of rows returned: if a page comes back full, there's probably a next page; if it comes back short, the next button is disabled automatically. No extra input needed.

The theming story

This is the part I'm actually most happy with.

Angular Material 3 uses CSS custom properties (system tokens like --mat-sys-primary) for everything. simple-table builds on top of that — all the visual properties are exposed as CSS variables you can override per-instance or globally:

simple-table {
  --st-row-bg: #f8fafc;
  --st-row-hover-bg: #e2e8f0;
  --st-header-bg: #1e293b;
  --st-header-color: #f1f5f9;
  --st-border-color: #cbd5e1;
  --st-selected-row-bg: #dbeafe;
}

There are about 22 tokens in total — row backgrounds, hover states, selection colours, header, border, sticky column background, cell colour. If you don't set any of them, they fall back to the Material 3 system tokens so they work with whatever theme you're already using.

Dark mode works automatically if you set color-scheme: dark on the body. The Material tokens switch, and any light-dark() values in your custom cell styles adapt with them. No JavaScript needed for the colour switching — just CSS.

Other bits worth mentioning

  • Column reordering — drag and drop headers to rearrange columns. CDK-powered. Sticky columns are excluded.
  • Column resizing — drag the right edge of any header to resize. Emits (columnWidthChange) so you can persist widths.
  • Column chooser — toolbar menu to show/hide and reorder columns.
  • State persistence — drop in stStateStoring and the component saves column order, visibility, and widths to localStorage automatically.
  • Export to Excel — stExport adds an export button. Provide an allDataProvider callback to fetch all records (not just the current page) for a full export. Uses exceljs (optional peer dep).
  • Sticky columns — sticky: 'left' or sticky: 'right' on a column def pins it during horizontal scroll.
  • Fill-container height — fillContainer: true stretches the table to fill its parent; toolbar and paginator always stay in view.
  • Client-side mode — pass a plain array and set clientSide: true in the config. The component handles sort, filter, and pagination internally with MatTableDataSource.

Installing it

npm install ngx-mat-simple-table

Peer deps are Angular 21+ and Angular Material 21+. exceljs is optional — only needed if you use the export feature.

Then import the components and directives you need:

import {
  SimpleTableComponent,
  CellDefDirective,
  StStateStoringDirective,
  StExportDirective,
} from 'ngx-mat-simple-table';
